  <action>
Create the two foundational modules that every subsequent file imports.

1. Create `src/config.ts`:
   - Import `z` from `'zod'`
   - Define `configSchema` as a nested `z.object` grouped by concern:
     - `telegram`: `z.object({ apiId: z.coerce.number().int().positive(), apiHash: z.string().min(1), session: z.string().min(1), botToken: z.string().min(1) })`
     - `claude`: `z.object({ apiKey: z.string().min(1), model: z.string().default('claude-sonnet-4-20250514') })`
     - `schedule`: `z.object({ activeHoursStart: z.coerce.number().int().min(0).max(23), activeHoursEnd: z.coerce.number().int().min(0).max(23), timezone: z.string().default('America/New_York') })`
     - `logLevel`: `z.enum(['debug', 'info', 'warn', 'error']).default('info')`
   - Export `type Config = z.infer<typeof configSchema>`
   - Export `function loadConfig(): Config` that:
     - Calls `configSchema.safeParse()` mapping `process.env.*` to the nested structure:
       - `telegram.apiId` from `process.env.TELEGRAM_API_ID`
       - `telegram.apiHash` from `process.env.TELEGRAM_API_HASH`
       - `telegram.session` from `process.env.TELEGRAM_SESSION`
       - `telegram.botToken` from `process.env.TELEGRAM_BOT_TOKEN`
       - `claude.apiKey` from `process.env.CLAUDE_API_KEY`
       - `claude.model` from `process.env.CLAUDE_MODEL`
       - `schedule.activeHoursStart` from `process.env.ACTIVE_HOURS_START`
       - `schedule.activeHoursEnd` from `process.env.ACTIVE_HOURS_END`
       - `schedule.timezone` from `process.env.TIMEZONE`
       - `logLevel` from `process.env.LOG_LEVEL`
     - On validation failure: format ALL issues as `"  - {path.join('.')}: {message}"` joined by newlines, print `"Config validation failed:\n{formatted}"` to stderr, call `process.exit(1)`
     - On success: return `result.data`

2. Create `src/logger.ts`:
   - Import `pino` from `'pino'` and `type { Config }` from `'./config'`
   - Declare module-level `let rootLogger: pino.Logger`
   - Export `function createLogger(config: Config): pino.Logger` that:
     - Creates Pino instance with:
       - `level`: `config.logLevel`
       - `transport`: only when `process.env.NODE_ENV === 'development'` -- set to `{ target: 'pino-pretty', options: { colorize: true } }`
       - `base`: `{ service: 'telegram-lead-bot' }`
       - `timestamp`: `pino.stdTimeFunctions.isoTime`
     - Assigns to `rootLogger` and returns it
   - Export `function getLogger(module: string): pino.Logger` that:
     - Throws `new Error('Logger not initialized. Call createLogger first.')` if `rootLogger` is undefined
     - Returns `rootLogger.child({ module })`

3. Verify the build compiles:
   - Create a minimal `src/index.ts` placeholder: `export {}` (just enough for tsup to build)
   - Run `npm run build` to confirm tsup produces `dist/index.js`
   - Run `npm run typecheck` to confirm no TypeScript errors
  </action>
