Create three files that wire the full application together.

1. Create `src/telegram.ts`:
   - Import `TelegramClient` from `'telegram'`
   - Import `StringSession` from `'telegram/sessions'`
   - Import `type { Config }` from `'./config'`
   - Import `type { Logger }` from `'pino'`
   - Export `async function connectTelegram(config: Config, logger: Logger): Promise<TelegramClient>`:
     - Create `new StringSession(config.telegram.session)`
     - Create `new TelegramClient(session, config.telegram.apiId, config.telegram.apiHash, { connectionRetries: 5, retryDelay: 1000, autoReconnect: true, deviceModel: 'MacBook Pro', systemVersion: '14.0', appVersion: '1.0.0', floodSleepThreshold: 60 })`
     - IMPORTANT: Set `deviceModel: 'MacBook Pro'`, `systemVersion: '14.0'`, `appVersion: '1.0.0'` -- realistic values to avoid Telegram's cloud IP anti-abuse (GramJS #773)
     - Call `await client.connect()`
     - Call `const me = await client.getMe()` and log: `logger.info({ event: 'telegram_connected', userId: me?.id?.toString() }, 'GramJS client connected')`
     - Return `client`
   - Export `async function disconnectTelegram(client: TelegramClient): Promise<void>`:
     - Call `await client.disconnect()`

2. Create `src/index.ts` (replace the placeholder from Plan 01):
   - Import `{ loadConfig }` from `'./config'`
   - Import `{ createLogger }` from `'./logger'`
   - Import `{ connectTelegram, disconnectTelegram }` from `'./telegram'`
   - Define `async function main()`:
     - Step 1: `const config = loadConfig()` -- validates all env vars, exits 1 on failure
     - Step 2: `const logger = createLogger(config)` -- initializes Pino
     - Step 3: `logger.info({ event: 'startup' }, 'Config validated, starting up')`
     - Step 4: `const client = await connectTelegram(config, logger)` -- connects GramJS
     - Step 5: `logger.info({ event: 'connected' }, 'Telegram client connected, process running')`
     - Step 6: Define `const shutdown = async (signal: string) => { logger.info({ event: 'shutdown', signal }, 'Received signal, shutting down'); await disconnectTelegram(client); logger.flush(); setTimeout(() => process.exit(0), 5000).unref(); }`
       - The 5-second timeout is a fallback for GramJS's hanging _updateLoop promise (issue #242, #615)
     - Step 7: `process.on('SIGTERM', () => shutdown('SIGTERM'))` and `process.on('SIGINT', () => shutdown('SIGINT'))`
   - BEFORE `main()` call, register global error handlers:
     - `process.on('uncaughtException', (err) => { console.error('Uncaught exception:', err); process.exit(1); })`
     - `process.on('unhandledRejection', (reason) => { console.error('Unhandled rejection:', reason); process.exit(1); })`
   - Call `main().catch((err) => { console.error('Fatal startup error:', err); process.exit(1); })`

3. Create `scripts/generate-session.ts`:
   - This is a one-time local script -- NEVER runs in production
   - Import `TelegramClient` from `'telegram'`
   - Import `StringSession` from `'telegram/sessions'`
   - Import `input` from `'input'`
   - Read `TELEGRAM_API_ID` and `TELEGRAM_API_HASH` from `process.env`
   - Validate both exist, exit 1 with message if not
   - Create `new StringSession('')` (empty = new session)
   - Create `new TelegramClient(session, apiId, apiHash, { connectionRetries: 5, deviceModel: 'MacBook Pro', systemVersion: '14.0', appVersion: '1.0.0' })`
   - Call `await client.start({ phoneNumber: async () => await input.text('Phone number (with country code): '), password: async () => await input.text('2FA password (if enabled): '), phoneCode: async () => await input.text('Code from Telegram: '), onError: (err) => console.error('Auth error:', err) })`
   - Print: `'\n--- Copy this StringSession to TELEGRAM_SESSION env var ---'`
   - Print: `client.session.save()`
   - Print: `'--- End of session string ---\n'`
   - Call `await client.disconnect()` then `process.exit(0)`

4. Verify everything builds:
   - Run `npm run build` -- must produce `dist/index.js`
   - Run `npm run typecheck` -- must exit 0
